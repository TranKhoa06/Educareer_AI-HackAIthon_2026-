\# Security Policy



\## Reporting a Security Issue



Please do not publicly disclose security vulnerabilities before the project maintainers have had an opportunity to review them.



When reporting a security issue, please include:



\- A clear description of the issue.

\- The affected feature, page, or file.

\- Steps to reproduce the issue.

\- The possible impact.

\- Suggested solutions, when available.



Please do not include real API credentials, private student information, or other sensitive data in public GitHub issues.



\## API Keys and Credentials



Never commit the following information to the repository:



\- `.env` files.

\- API keys.

\- Access tokens.

\- Token IDs.

\- Token keys.

\- Passwords.

\- Private service credentials.



Use `.env.example` only to provide placeholder variable names.



All real credentials should be stored locally or configured through secure deployment environment variables.



\## Student Information



EduCareer AI may process educational information such as academic scores, career interests, personality-assessment results, and competency information.



Contributors should use fictional, anonymized, or sample information during development and testing.



Do not upload:



\- Real academic transcripts without permission.

\- Student identification information.

\- Private assessment results.

\- Personal contact information.

\- Private educational records.



\## Data Privacy Principles



EduCareer AI aims to follow these principles:



\- Collect only information required for the requested feature.

\- Avoid unnecessary storage of student information.

\- Minimize the use of personally identifiable information.

\- Explain how student information is processed.

\- Protect private credentials and sensitive data.

\- Avoid exposing personal information through logs, screenshots, or public repositories.



\## Responsible AI



AI-generated career recommendations should be treated as educational guidance rather than final decisions.



The platform should:



\- Explain the reasons behind recommendations.

\- Avoid guaranteeing academic or career outcomes.

\- Acknowledge uncertainty when student information is incomplete.

\- Avoid harmful stereotypes and unfair assumptions.

\- Avoid permanently labeling students.

\- Encourage students to consult teachers, parents, career counselors, and reliable educational sources.



\## Supported Version



The latest version of the `main` branch is currently supported.



EduCareer AI is an early-stage open-source project. Security and privacy practices will continue to be reviewed and improved as the project develops.
